# Phase Outputs

Each subdirectory contains the deliverables produced by executing that phase of the master plan. The plan itself (`plan/surf_betting_master_plan_v2.md`) contains only instructions — all findings, data, analysis, and decisions go here.

## Structure

- `phase_0/` — Domain reference document, decision log, scenario model, assumptions register
- `phase_1/` — Data source inventory, gap analysis, walled-garden flags
- `phase_2/` — ERD, ETL diagram, tech stack decisions
- `phase_3/` — Pilot data, DQ report, known-answer test results
- `phase_4/` — Full dataset, data dictionary, DQ report
- `phase_5/` — Feature set documentation, visualizations, edge map, calibration models
- `phase_6/` — Model comparison report, Elo/Glicko-2 design decisions
- `phase_7/` — Trained models, diagnostics, calibration plots
- `phase_8/` — Validation report, baseline comparisons, error analysis
- `phase_9/` — Betting strategy research, staking rules, platform strategy
- `phase_10/` — Backtest results, Monte Carlo simulations, sensitivity analysis
- `phase_11/` — Paper trading log, execution gap analysis
- `phase_12/` — Live performance tracking, platform health monitoring
- `phase_13/` — Drift detection reports, ALT reconstruction updates, retraining logs
